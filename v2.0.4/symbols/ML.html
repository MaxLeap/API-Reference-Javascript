<static> ML.Events

ML.Events is a fork of Backbone's Events module, provided for your convenience.

A module that can be mixed in to any object in order to provide it with custom events. You may bind callback functions to an event with `on`, or remove these functions with `off`. Triggering an event fires all callbacks in the order that `on` was called. 

    var object = {};
    _.extend(object, ML.Events);
    object.on('expand', function(){ alert('expanded'); });
    object.trigger('expand');

For more information, see the Backbone documentation.

<static> ML.Op()
A ML.Op is an atomic operation that can be applied to a field in a ML.Object. For example, calling object.set("foo", "bar") is an example of a ML.Op.Set. Calling object.unset("foo") is a ML.Op.Unset. These operations are stored in a ML.Object and sent to the server as part of object.save() operations. Instances of ML.Op should be immutable. You should not create subclasses of ML.Op or instantiate ML.Op directly.
